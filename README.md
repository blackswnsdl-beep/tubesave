## v4 — 상용앱 편의 기능 5종 (설계 검토 후 구현)
설계 검토 문서: 기능추가_설계검토_v4.md (팀 에이전트 + 악마의 대변인 논의·합의)
- **백업/복원**: 앨범·즐겨찾기·바꾼 이름을 파일로 저장/복원(노래 파일은 음악 폴더에 그대로). 복원 전 확인, 깨진 파일이면 기존 데이터 보존.
- **첫 실행 사용법 안내**: 처음 켜면 1회, 설정에서 다시 보기.
- **중복 다운로드 안내**: 같은 제목이 이미 있으면 저장 안 하고 "이미 있어요" — 중복 누적 방지.
- **잠금화면 앨범 커버**: 다운로드 시 1회 썸네일 추출 → 재생 시 잠금화면/알림에 표시(삭제 시 함께 정리). ※실기기 표시 검증 필요.
- **홈 화면 위젯**: 제목 + 이전/재생·정지/다음. MediaController 경유 제어로 백그라운드 크래시 회피, try/catch 격리. ※미검증 위험 가장 큼 → 빌드 후 1순위 테스트.

빌드 후 테스트 체크리스트는 설계검토 문서 5번 참고.

## v3.7 — 다운로드가 막히면 자동 업데이트 안내
- 노래 받기가 **2번 연속 실패**하면 "노래가 잘 안 받아져요. 지금 업데이트할까요?" 팝업이 자동으로 뜸.
- 앱이 켜져 있으면 화면에 다이얼로그로, 꺼져 있으면 실패 알림을 누르면 같은 팝업이 뜸(공유로 받다 실패한 경우도 커버).
- [지금 업데이트]를 누르면 진행 표시 후 yt-dlp를 최신(NIGHTLY)으로 갱신 → 결과 안내. 어르신은 버튼 한 번이면 됨.
- escalation 구조: 평소엔 실패 시 조용히 STABLE 자동 갱신(기존) → 막히면 더 적극적인 NIGHTLY 수동 갱신.
- 팝업 중복 방지(여러 곡 실패 시 다이얼로그 안 겹침), 인터넷 없으면 그대로 알림.

## v3.6 — 상용앱 수준 편의 기능 3종
- **이어듣기**: 앱을 닫았다 켜면 마지막 재생목록·곡·재생 위치를 복원(준비 상태로 둠, 자동재생은 안 함 — 재생 버튼만 누르면 이어짐). 일시정지/앱 종료 시 위치를 저장. 잠금화면에서 멈춘 위치도 반영.
- **곡 이름 바꾸기**: 유튜브 제목이 지저분할 때 보기 쉬운 이름으로. 파일은 그대로 두고 표시 이름만 저장(앱 전용). 전체 곡에서 1곡만 선택하면 [이름변경] 버튼이 나옴. 이름을 비우면 원래대로 복귀.
- **정렬**: 전체 곡을 최신순/이름순으로(설정에서 전환, 즐겨찾기는 항상 위). 한글 가나다순 정상 정렬.
- 모두 순수 로직(SharedPreferences + 목록 처리)이라 기기 호환성 이슈 없음. 곡 전환마다 전체 목록을 다시 그리지 않는 최적화 유지.

## v3.5 — 앨범(곡 모음) 기능
- **여러 앨범 만들기**: 좋아하는 곡들을 주제별로 묶어 앨범으로 관리. 탭이 [전체 곡] / [앨범]으로 구성됨.
- **앨범 관리**: 추가 / 이름 변경 / 삭제 / 순서 이동(▲▼·드래그). 앨범 삭제해도 노래 파일은 남음.
- **앨범 상세**: 전체재생, 곡 탭하면 그 지점부터 연속재생, 곡 순서 이동, 곡 빼기.
- **곡 추가**: 전체 곡에서 [선택] → [앨범에 추가] → 기존 앨범 선택 또는 새 앨범 만들기. 같은 곡은 자동으로 건너뜀.
- **동기화**: 파일을 삭제하면 모든 앨범에서도 자동 정리(라이브러리 대조로 빠르게 처리, IO 최소화).
- 단일 "재생목록"은 다중 "앨범"으로 대체됨(PlaylistRepository → AlbumRepository).


## v3.4 — 전반 점검 (성능·보안·견고성)
- **성능(최우선)**: 곡이 바뀔 때마다 목록 전체를 재정렬·재바인딩하던 동작 제거 → 바뀐 행만 갱신. 재생 중 목록이 정적이고 매끄러움. 재생시간 캐시로 목록 로딩도 빠름.
- **보안**: allowBackup=false (저장된 텔레그램 봇 토큰이 백업으로 유출되지 않음). 다운로드는 youtube 도메인만 허용하고 yt-dlp에 인자로 전달(쉘 인젝션 불가). 파일 삭제는 앱 폴더 내로 한정. 완료 브로드캐스트는 앱 내부 전용.
- **견고성**: Android 14에서 다운로드 포그라운드 서비스 타입(dataSync) 명시 → 시작 시 크래시 방지.
- **검증**: 리소스 참조 44/44, 레이아웃별 id 일치, gradle 의존성 일치 확인.

# 노래저장 (TubeSave)

유튜브 공유하기 → 앱 탭 → mp3 자동 저장 → 앱에서 바로 재생.
어르신용으로 설계된 초간단 유튜브 음악 다운로드 + 플레이어 앱.

## 주요 기능
- **공유하기 한 번으로 다운로드**: 유튜브 → 공유 → "노래저장" 탭. 화면 안 뜨고 알림으로 진행률 표시
- **링크 붙여넣기**: 앱 안에서 링크 붙여넣고 [받기] 버튼 (폴백 경로)
- **자동 mp3 변환**: 최고 음질, 앨범아트/제목 태그 자동 삽입
- **저장 폴더**: 기본은 `음악/TubeSave` (모든 음악앱에서 보임). 우상단 폴더 버튼으로 원하는 폴더 지정 가능
- **내장 플레이어**: 곡 목록 탭하면 그 곡부터 연속 재생. 큰 재생/이전/다음 버튼
- **잠금화면 재생**: 화면 꺼져도 라디오처럼 계속 재생. 잠금화면/알림에서 컨트롤 가능. 전화 오면 자동 일시정지
- **파일 관리**: 곡을 길게 누르면 선택 모드 → 체크박스 / 전체선택 / 선택 삭제 (Android 11+는 시스템 확인창 연동)
- **재생시간 표시**: 곡마다 분:초 표시
- **선택 재생**: 여러 곡 체크 후 [재생] → 선택한 곡들 순차 재생
- **재생목록 탭**: 선택 곡 [목록추가]로 등록, 드래그 핸들로 순서 변경, [목록에서 빼기]로 제거 (파일은 유지)

## 빌드 방법 (Android Studio)
1. Android Studio 설치 (최신 버전, JDK 17 포함됨)
2. `File → Open` 으로 이 폴더(TubeSave) 열기 → Gradle 동기화 자동 진행
3. `Build → Build App Bundle(s) / APK(s) → Build APK(s)`
4. 생성 위치: `app/build/outputs/apk/debug/app-debug.apk`

### 명령줄 빌드 (Android SDK 설치된 환경)
```bash
./gradlew assembleDebug
```

## 어르신 폰에 설치하기
1. `app-debug.apk` 를 카톡/USB로 폰에 전송
2. 파일 탭 → "출처를 알 수 없는 앱 설치 허용" 한 번 켜기 → 설치
3. 앱 최초 실행 시 알림/오디오 권한 [허용] 눌러주기
4. 유튜브에서 노래 → 공유 → **노래저장** 탭 → 끝

## 유지보수 참고
- 유튜브가 내부 구조를 바꾸면 다운로드가 실패할 수 있음.
  앱이 실행 시 yt-dlp 코어를 자동 업데이트하도록 되어 있어 대부분 자동 해결되지만,
  장기간 실패 시 `youtubedl-android` 라이브러리 버전을 올려 APK 재빌드 필요.

- 개인 사용 목적으로만 사용할 것 (유튜브 약관상 콘텐츠 다운로드는 제한됨)


## v3 변경점
- **반복 모드 3단계 토글**: 재생바 좌측 버튼으로 [↻ 전체반복] → [↺ 한곡반복] → [→ 반복없음] 순환. 설정 유지. 기본값은 전체반복(라디오처럼 멈추지 않음)
- **상시 [선택] 버튼**: 탭 줄 오른쪽에 항상 표시. 길게 누르기를 몰라도 누구나 선택 모드 진입 가능 (기존 길게 누르기도 보조로 동작)
- **다운로드 직렬 큐**: 여러 곡을 연달아 공유해도 하나씩 순서대로 처리. "N곡 중 M번째" 통합 알림. 요청별 임시폴더 분리로 충돌/유실 제거
- **안전한 액션바**: 삭제 버튼을 2행 오른쪽으로 격리(빨강), 취소와 분리해 오탭 삭제 방지
- **재생목록 ▲▼ 보조 버튼**: 드래그 외에 위/아래 버튼으로 한 칸씩 순서 이동. 손 떨림이 있어도 정확히 이동 가능(드래그는 그대로 유지)
- **진행 표시줄(seek bar)**: 현재 위치/전체 길이(분:초) 표시, 막대를 끌어 원하는 지점으로 이동
- **탭 시각 강조**: 선택된 탭은 채운 주황 알약, 비선택은 회색으로 명확히 구분
- **취침 타이머**: 설정에서 15/30/60분 후 자동 정지 (잠들 때 라디오처럼)
- **셔플**: 설정에서 켜기/끄기 (메인 UI는 그대로 단순 유지)
- **재생시간 캐시**: 곡이 많아도 목록 로딩이 빠름 (수정시각 기준 캐시)
- **죽은 항목 자동 정리**: 외부에서 파일을 지워도 재생목록이 깨지지 않음
- **실패 안내**: 다운로드 실패 시 비기술적 안내. 반복 실패 시 '앱 업데이트 필요' 안내. 관리자는 설정에서 텔레그램 봇 토큰/챗ID를 넣으면 실패 알림을 받음
- **통합 설정**: 헤더 버튼 한 곳에 저장 폴더·취침 타이머·셔플·알림설정을 모음
- **디자인 통일**: 따뜻한 앰버 컬러 토큰, 둥근 버튼, 원형 재생 버튼, 흰색 벡터 아이콘으로 시각적 일관성 확보. 대비비(WCAG AA) 고려해 브랜드색을 #E65100으로 조정


## v3.3 (P2 + 코드 감사)
**새 기능**
- **즐겨찾기**: 곡 오른쪽 별을 눌러 등록. 즐겨찾기 곡은 전체 곡 목록 맨 위로 모임
- **검색**: 상단 돋보기 → 제목으로 곡 찾기 (전체 곡/재생목록 모두). 기본은 숨김이라 평소 화면은 단순
- **큰 재생 화면**: 하단 곡 제목을 누르면 전체화면 플레이어(큰 음표·큰 버튼·진행바). 메인 컨트롤러를 그대로 써서 추가 연결 없음

**버그 수정**
- 취침 타이머가 화면 잠금 후 작동하지 않던 문제 → 재생 서비스로 옮겨 Activity와 무관하게 동작
- 목록 로딩의 메인 스레드 I/O(곡 많을 때 끊김) → 전부 백그라운드 처리
- 드래그 정렬 중 매번 디스크에 쓰던 문제 → 드래그 종료 시 1회만 저장

**구조/유지보수**
- 설정 접근을 `Prefs` 한 곳으로 통합(키 문자열 중복 제거, 확장 용이)
- 검색 중에는 순서 변경을 막아 인덱스 꼬임 방지

**알려진 한계**
- 목록 갱신은 `notifyDataSetChanged` 사용(곡 수십~수백 기준 체감 영향 없음). 수천 곡 규모가 되면 DiffUtil 도입 권장
- 관리자 텔레그램 토큰은 앱 전용 저장소에 평문 저장(개인 유지보수용으로 한정). 민감도가 높아지면 EncryptedSharedPreferences로 교체 가능
